## TDD 가이드라인

### 레드-그린-리팩토링 사이클
1. **레드**: 실패하는 테스트 작성
2. **그린**: 최소한의 코드로 테스트 통과
3. **리팩토링**: 코드 개선 및 중복 제거

### 테스트 작성 순서
1. **단위 테스트**: 순수 함수 로직
2. **통합 테스트**: 여러 함수 조합
3. **사용자 테스트**: 실제 사용 시나리오

### 커밋 메시지 규칙
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `test`: 테스트 관련
- `style`: 스타일 변경
- `perf`: 성능 최적화
- `deploy`: 배포 관련
